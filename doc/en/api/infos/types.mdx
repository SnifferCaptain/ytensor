# Type Registration

This document introduces the type registration system in the `yt::infos` namespace.

## TypeRegItem Struct

```cpp
struct TypeRegItem {
    std::string name;                              // Type name
    int32_t size;                                  // Size in bytes
    std::function<std::string(const void*)> toString;  // Format to string
    bool isPOD = true;                             // Is POD type
    std::function<void(void*)> destructor = nullptr;   // Destructor
    std::function<void(void*, const void*)> copyConstruct = nullptr;  // Copy constructor
    std::function<void(void*)> defaultConstruct = nullptr;  // Default constructor
    
    // IO Serialization Support (for non-POD types)
    std::function<std::vector<char>(const void*)> serialize = nullptr;      // Object -> Byte Sequence
    std::function<void(void*, const char*, size_t)> deserialize = nullptr;  // Byte Sequence -> Object
};
```

## Type Registry

```cpp
auto& getTypeRegistry();
```

Returns a reference to the global type registry.

### Registering Types

In addition to automatically deduced POD types, YTensor provides the `registerType` function for registering custom types.

#### 1. Basic Registration (Auto POD Detection)

```cpp
template<typename T>
void registerType(const std::string& typeName, 
                  std::function<std::string(const void*)> formatter = nullptr);
```

#### 2. Registration with IO Support (Non-POD)

```cpp
template<typename T>
void registerType(const std::string& typeName,
                  std::function<std::string(const void*)> formatter,
                  std::function<std::vector<char>(const void*)> serialize,
                  std::function<void(void*, const char*, size_t)> deserialize);
```

This overload allows you to provide serialization and deserialization functions for non-POD types (such as `std::string` or custom objects), enabling them to be correctly stored and loaded by `YTensorIO`.

## Floating Point Types Specification

YTensor supports various floating point types, including standard and low-precision floats.
Specifically, for standard `float16` and `bfloat16`, the rounding mode is affected by global configuration (see [Constants](./constants.mdx)).

## FloatSpec Template

YTensor provides the `FloatSpec` template class to generate arbitrary floating point types based on IEEE 754 standard configuration.

### Template Definition

```cpp
template <int SignBits, int ExpBits, int MantissaBits>
class FloatSpec;
```

| Parameter | Description |
| --- | --- |
| `SignBits` | Number of sign bits (0 or 1) |
| `ExpBits` | Number of exponent bits |
| `MantissaBits` | Number of mantissa bits |

### Predefined Types

These types are predefined in the library and can be used directly:

| Type Name | C++ Alias | Definition | Total Bits | Description |
| --- | --- | --- | --- | --- |
| `float16` | `yt::float16` | `FloatSpec<1, 5, 10>` | 16 | IEEE 754 Half Precision |
| `bfloat16` | `yt::bfloat16` | `FloatSpec<1, 8, 7>` | 16 | Brain Floating Point |
| `float8_e5m2` | `yt::float8_e5m2` | `FloatSpec<1, 5, 2>` | 8 | Hopper Architecture FP8 |
| `float8_e4m3` | `yt::float8_e4m3` | `FloatSpec<1, 4, 3>` | 8 | Hopper Architecture FP8 |
| `float8_e8m0` | `yt::float8_e8m0` | `FloatSpec<0, 8, 0>` | 8 | Pure Exponent (Scaling Factor) |

### Usage Example

```cpp
#include "ytensor_types.hpp"

// Use predefined types
yt::bfloat16 bf = 3.14f;
float val = static_cast<float>(bf);

// Custom floating point type (simulated TF32)
using TF32 = yt::FloatSpec<1, 8, 10>;
TF32 tf = 2.5f;
```

## Dtype Naming Convention

YTensor uses a normalized dtype naming format to support nested tensor types:

### Base Type Names
Use `getBaseTypeName<T>()` to get the base C++ type name:
- `float` → `"float32"`
- `double` → `"float64"`
- `int32_t` → `"int32"`
- Custom Type → `typeid(T).name()` or registered name

### Nested Type Format

| Tensor Type | dtype Format | Example |
| --- | --- | --- |
| `YTensor<T, dim>` | `"YTensor<scalar_dtype, dim>"` | `"YTensor<float32, 2>"` |
| `YTensorBase` | `"YTensorBase<inner_dtype>"` | `"YTensorBase<float32>"` |
| Multi-level Nesting | Recursive Format | `"YTensor<YTensor<float32, 2>, 2>"` |

### Dtype Parsing Functions

| Function | Description | Example |
| --- | --- | --- |
| `parseDtypeInner(dtype)` | Parse outer type and inner content | `"YTensorBase<float32>"` → `{"YTensorBase", "float32"}` |
| `getBaseDtype(dtype)` | Recursively extract innermost base type | `"YTensor<YTensor<float32, 2>, 2>"` → `"float32"` |
| `getDtypeDim(dtype)` | Extract YTensor dimension | `"YTensor<float32, 3>"` → `3` |
| `getDtypeScalar(dtype)` | Extract YTensor scalar type | `"YTensor<float32, 2>"` → `"float32"` |
| `makeYTensorDtype(scalar, dim)` | Build YTensor dtype | `("float32", 2)` → `"YTensor<float32, 2>"` |
| `makeYTensorBaseDtype(inner)` | Build YTensorBase dtype | `("float32")` → `"YTensorBase<float32>"` |

### Usage Example

```cpp
#include "ytensor_types.hpp"

// Parse nested dtype
std::string dtype = "YTensorBase<YTensor<float32, 2>>";
std::string baseDtype = yt::types::getBaseDtype(dtype);  // "float32"

// Build dtype
std::string ytensorDtype = yt::types::makeYTensorDtype("float32", 3);  // "YTensor<float32, 3>"

// Use with dispatch
YTensorBase tensor({2, 3, 4}, "float32");
auto matView = tensor.matView();  // dtype: "YTensorBase<float32>"

// Dispatch automatically parses nested types
yt::kernel::dispatch<yt::types::AllNumericTypes>(matView.dtype(), [&]<typename T>() {
    // T is parsed as float
});
```

## Non-POD Type Support

For non-POD types (like `std::string`, custom classes), registration is required before use. Destructors and constructors are automatically deduced during registration.

### Enabling IO Support

By default, `registerType` only registers basic lifecycle functions. If you need file IO support (saving to `.yt` files), you must explicitly provide `serialize` and `deserialize` functions.

```cpp
struct MyStruct {
    std::string data;
};

// Register and provide IO support
yt::types::registerType<MyStruct>(
    "MyStruct",
    // formatter
    [](const void* ptr) { return static_cast<const MyStruct*>(ptr)->data; },
    // serialize
    [](const void* ptr) -> std::vector<char> {
        const auto& s = static_cast<const MyStruct*>(ptr)->data;
        return std::vector<char>(s.begin(), s.end());
    },
    // deserialize
    [](void* ptr, const char* data, size_t len) {
        new (ptr) MyStruct{std::string(data, len)};
    }
);
```

:::info
**Storage Format**
For types with registered serialization functions, YTensorIO uses the `map` format for storage, ensuring variable-length data and object states are handled correctly.
:::
