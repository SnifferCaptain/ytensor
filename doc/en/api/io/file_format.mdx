# File Format (.yt)

This document details the binary file format specification for YTensor (`.yt`).

## 1. Overview

`.yt` is a binary format designed for efficient tensor storage. Its design goals are:
*   **Sequential Writing**: Tensors can be continuously appended to the file.
*   **Fast Indexing**: An index table at the end of the file allows O(1) random access to any tensor.
*   **Type Safety**: Stores complete shape, stride, and type information.

## 2. File Structure

A standard `.yt` file consists of three parts:

```
+-----------------------------------+
| 1. Global Header (Magic + Ver)    | Fixed 16 Bytes
+-----------------------------------+
| 2. Data Area (Tensor Records)     | Variable Length
|    [Tensor 1]                     |
|    [Tensor 2]                     |
|    ...                            |
+-----------------------------------+
| 3. Index Table                    |
|    [Index Entry 1]                |
|    ...                            |
|    [Index Table Footer]           | Fixed Size
+-----------------------------------+
```

---

## 3. Global Header

Fixed at the start of the file.

| Offset | Type | Value | Description |
| --- | --- | --- | --- |
| 0x00 | `char[8]` | `"YTENSOR!"` | Magic Number |
| 0x08 | `uint32` | `1` | Format Version (Major) |
| 0x0C | `uint32` | `0` | Header padding / Reserved |

---

## 4. Tensor Record

Each tensor is stored as a record containing a header and payload.

| Field | Type | Description |
| --- | --- | --- |
| **Magic** | `uint32` | `0xF00DCAFE` (Start Marker) |
| **Name Len** | `uint32` | Length of name string |
| **Name** | `char[]` | Tensor Name (UTF-8) |
| **Dtype Len** | `uint32` | Length of dtype string |
| **DType** | `char[]` | Data Type (e.g., "float32") |
| **NDim** | `uint32` | Number of dimensions |
| **Shape** | `uint64[N]` | Shape array (Length NDim) |
| **Stride** | `uint64[N]` | Stride array (Length NDim) |
| **Type** | `uint8` | TensorType (0=Dense, 1=Sparse, 2=Map) |
| **Comp** | `uint8` | Compression Flag (0=None, 1=Zlib) |
| **Data Size** | `uint64` | Size of Payload (Bytes, compressed size) |
| **Payload** | `byte[]` | Data Content (See Section 5) |

---

## 5. Payload Layout

The `Data` part of the tensor record (after decompression) has different layouts based on `TensorType`:

### 5.1 Dense Type (Default)

Standard dense tensor.
Content: **Row-Major (C-Order)** raw bytes of all elements.

*   **Total Bytes** = Total Elements × ElemSize
*   **Content**: `[Elem(0,0,...)] [Elem(0,0,...1)] ...`

### 5.2 Sparse Type (COO)

Sparse tensor. Data contains two parts: Indices and Values.
Storage format serialized as:

1.  **Indices**: `array[uint64_t]` (Flattened coordinate indices)
2.  **Values**: `array[T]` (Values of non-zero elements)

Here `ElemSize` still refers to size of T.

### 5.3 Map Type

Used for storing non-POD objects (like string lists). `ElemSize` is fixed to `TypeSize` (usually determined by type).

Data content serialized as (packed, no extra length prefix):

1.  **Offsets**: `uint64_t[N]`
    *   Start offset of the first N elements (relative to start of Values).
    *   Length of i-th element can be calculated by `offsets[i+1] - offsets[i]` (last element ends at data end).
2.  **Values**: Byte stream (Concatenation of all serialized object data)

**Note**: N equals total elements of the tensor, derived from `Shape` in Header. When reading, calculate N from Shape first, read N*8 bytes of Offsets, and the remainder is Values.

## 6. Implementation Reference (Pseudo-code)

Any language parser should follow similar logic:

```python
def read_yt_file(path):
    f = open(path, "rb")
    
    # 1. Read Footer (Last 16 bytes: Index Table Offset + Count)
    f.seek(-16, 2)
    index_offset = read_uint64(f)
    index_count = read_uint64(f)
    
    # 2. Jump to Index Table
    f.seek(index_offset)
    indexes = []
    for _ in range(index_count):
        indexes.append(read_index_entry(f))
        
    # 3. Read specific tensor
    target = indexes["my_tensor"]
    f.seek(target.file_offset)
    tensor = read_tensor_record(f)
    
    return tensor
```
