# YTensorIO

The `yt::io::YTensorIO` class provides functionality for reading and writing `.yt` files.

## Overview

```cpp
#include "ytensor_io.hpp"

// Create IO object (automatically opens file, RAII)
yt::io::YTensorIO io("data.yt", yt::io::Mode::Write);

// Write
io.save(tensor, "tensor_name");

// IO object is automatically closed when it goes out of scope, or manually call close()
io.close();
```

---

## `Mode` Enum

File opening modes:

| Mode | Description | Behavior |
| --- | --- | --- |
| `Read` | Read-only | Default mode. Error if file does not exist. |
| `Write` | Write-only | Creates new file. If exists, **overwrites completely**. |
| `Append` | Append | Opens existing file to append data. Error if file does not exist. |

---

## `save()`

Writes a tensor to a file.

```cpp
void save(const yt::YTensorBase& tensor, 
          const std::string& name, 
          bool compress = true);
```

### Parameters

*   `tensor`: The tensor reference to save.
*   `name`: Name of the tensor in the file (must be unique).
*   `compress`: Whether to enable Data Payload compression (default `true`). The index area is always uncompressed.

Warning: Enabling compression saves space but increases CPU usage. For speed-sensitive scenarios (e.g., buffering), consider setting it to `false`.

### Usage Example

```cpp
YTensor<float, 2> matrix({100, 100});

yt::io::YTensorIO io("model.yt", yt::io::Mode::Write);
io.save(matrix, "weight_1");
// You can save multiple tensors
io.save(bias, "bias_1");
```

### Notes

*   Must be called in `yt::io::Write` or `yt::io::Append` mode.
*   Tensor names cannot repeat existing names in the file (especially in Append mode).

:::info
**Non-POD Type Support**
Starting from v0.1, YTensor IO supports storage of non-POD types (like `std::string`).
This requires the type to provide `serialize` and `deserialize` callback functions during registration. See [Type Registration](../infos/types.mdx) for details.

**Note**: Due to current implementation limits, the data size generated by `serialize` should not be significantly larger than the object's memory size (`sizeof(T)`), otherwise reading may fail due to insufficient buffer pre-allocation. It is recommended to store only short strings or small structures.
:::

## `load()`

Reads a tensor from a file.

```cpp
void load(const std::string& name, yt::YTensorBase& tensor);
// Or overload returning new object
YTensorBase load(const std::string& name);
```

### Parameters

*   `name`: Name of the tensor to read.
*   `tensor`: Output tensor object (will be resized).

### Usage Example

```cpp
yt::io::YTensorIO io("model.yt", yt::io::Mode::Read);

// Method 1: Load into existing object
YTensorBase t1;
io.load("weight_1", t1);

// Method 2: Return new object
auto t2 = io.load("bias_1");
```

### Notes

*   Must be called in `yt::io::Read` or `yt::io::Append` mode.
*   If `name` does not exist, throws `std::runtime_error`.

---

## `alloc_buffer()`

Pre-allocates serialization buffer. Optional optimization.

```cpp
void alloc_buffer(size_t size);
```

If you anticipate saving very large tensors (e.g., >100MB), invoking this beforehand can reduce memory reallocation overhead.

---

## `close()`

Manually closes the file.

```cpp
void close();
```

*   In `Write/Append` mode: Calling `close()` will **write the Index Table** to the end of the file. If the program crashes before `close()` (destructor not called), the file may be corrupted (missing index table).
*   `YTensorIO` automatically calls `close()` in its destructor unless manually called.

## Static Helper Functions

### `save_to_file`

Convenience function for faster one-time saving.

```cpp
static void save_to_file(const yt::YTensorBase& tensor, 
                         const std::string& filepath, 
                         const std::string& name = "tensor");
```

### `load_from_file`

Convenience function for faster one-time loading.

```cpp
static YTensorBase load_from_file(const std::string& filepath, 
                                  const std::string& name = "tensor");
```
