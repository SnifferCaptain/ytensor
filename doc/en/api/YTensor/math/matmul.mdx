# Matrix Multiplication (matmul)

```cpp
YTensor<T, 2> matmul(const YTensor<T, 2>& other) const;
```

Performs matrix multiplication: `Result = (*this) @ other`.

## Rules

1.  Input must be 2D matrices (or viewable as 2D).
2.  Shapes must satisfy `(M, K) x (K, N) -> (M, N)`.
3.  Supports mixed Types (auto-promotion).

## Backend Implementation

YTensor automatically selects the optimal backend based on hardware and compilation options:

| Backend | Condition | Features |
| --- | --- | --- |
| **AVX2** | `YT_USE_AVX2` + x86 CPU | Blocked optimization, SIMD, max performance. **Requires Row-Major inputs!** |
| **Eigen** | `YT_USE_EIGEN` | Uses Eigen library. High performance, handles non-contiguous well. |
| **Naive** | Default | Slow, pure C++ loops. For reference/debugging. |

## Usage

```cpp
auto A = YTensor<float, 2>::rand(100, 50);
auto B = YTensor<float, 2>::rand(50, 20);

auto C = A.matmul(B); // Shape [100, 20]
```

## Performance Note

If using the **AVX2** backend, ensure input matrices are stored in **Row-Major** order (Contiguous) for best performance. If inputs are Transposed (Column-Major), the backend might need to perform an internal copy/repacking, adding overhead.

```cpp
// AVX2 Prefer:
A.matmul(B);

// Might be slower due to memory layout conversion:
A.transpose().matmul(B); 
```
