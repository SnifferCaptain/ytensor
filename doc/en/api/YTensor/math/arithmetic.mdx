# Arithmetic Operations

YTensor supports rich arithmetic operator overloading.

## Basic Operators

Supports `+`, `-`, `*`, `/`, `%` (modulo).

```cpp
auto a = YTensor<float, 2>::ones(2, 2);
auto b = YTensor<float, 2>::ones(2, 2) * 2.0f;

auto c = a + b;  // Element-wise addition
auto d = a * b;  // Element-wise multiplication (Hadamard product)
auto e = a / 2.0f; // Scalar division
```

**Note**: `*` is element-wise multiplication. For matrix multiplication, use `matmul`.

### Modulo Operator `%`

*   For Integers: Standard modulo.
*   For Floats: Uses `fmod`.

## In-Place Operators

`+=`, `-=`, `*=`, `/=`, `%=`.

```cpp
a += b;      // a = a + b
a *= 3.0f;   // Scale a by 3
```

**Warning**:
1.  In-place operations modify the data. Any other views sharing this data will see the change.
2.  In-place operations generally assume shape consistency (or broadcastable, depending on implementation).

## Broadcasting

Arithmetic operations support NumPy-style broadcasting rules.

1.  Align dimensions from the right.
2.  If dimension sizes differ, one of them must be 1.
3.  The dimension of size 1 is conceptually stretched to match the other.

```cpp
// shape [2, 3]
auto m = YTensor<float, 2>::ones(2, 3);
// shape [3]
auto v = YTensor<float, 1>::ones(3);

// v is broadcasted to [1, 3] then [2, 3]
auto res = m + v;
```

## Comparison Operators

`==`, `!=`, `<`, `<=`, `>`, `>=` return a boolean tensor (`YTensor<bool, D>`).

```cpp
auto mask = a > 0.5f;
```

## Bitwise Operators

`<<`, `>>` (Shift) are supported for integer types.
