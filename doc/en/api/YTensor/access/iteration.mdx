# Iteration

Efficiently iterating over tensor elements.

## 1. Using `foreach()` (Recommended)

`foreach` automatically selects the optimal traversal strategy (parallel/serial) and handles non-contiguous memory correctly.

```cpp
auto t = YTensor<float, 2>::rand(100, 100);

// Simple traversal
t.foreach([](float& val) {
    val += 1.0f;
});

// With coordination
t.foreach([](float& val, const std::vector<int>& pos) {
    if (pos[0] == pos[1]) { // Diagonal
        val = 1.0f;
    }
});
```

*   **Performance**: Automatically parallelized (OpenMP-like).

## 2. Using `begin()` / `end()` (STL Style)

YTensor provides standard iterators compatible with STL algorithms.

```cpp
// Range-based for (Valid only for contiguous tensors?)
// Note: Currently YTensor iterators iterate over physical memory.
// Use with caution on non-contiguous views.
for (auto& val : t) {
    val *= 2.0f;
}

// STL Algorithms
std::fill(t.begin(), t.end(), 0.0f);
```

## 3. High Performance Pointer Iteration

For **contiguous** tensors, raw pointer access is the absolute fastest.

```cpp
if (t.isContiguous()) {
    float* ptr = t.data();
    size_t n = t.size();
    
    #pragma omp parallel for
    for (size_t i = 0; i < n; ++i) {
        ptr[i] += 1.0f;
    }
} else {
    // Fallback requires coordinate calculation
    t.foreach(...)
}
```
