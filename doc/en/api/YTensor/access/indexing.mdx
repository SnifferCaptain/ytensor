# Indexing

YTensor provides multiple ways to access elements.

## Recommendations

| Method | Syntax | Boundary Check | Coordinate Conversion | Performance | Scenario |
| --- | --- | --- | --- | --- | --- |
| **at()** | `at(i, j)` | ❌ No | ✅ Auto | Low | Random access to non-contiguous tensors |
| **atData()** | `atData(idx)` | ❌ No | ✅ Auto | Low | Linear logic index access |
| **atData_()** | `atData_(idx)` | ❌ No | ❌ None | **High** | Iterating **contiguous** tensors data |
| **operator[]** | `[idx]` | ❌ No | ❌ None | **High** | Same as `atData_`, C-style array compatible |

## 1. Multi-dimensional Coordinator Access (`at`)

Access elements using natural multi-dimensional coordinates.

```cpp
auto t = YTensor<float, 2>::zeros(3, 4);

// Write
t.at(0, 1) = 3.14f;

// Read
float val = t.at(0, 1);
```

*   **Pros**: Intuitive, works correctly on any view (transposed, sliced).
*   **Cons**: Slower due to coordinate-to-offset calculation required for every access.

## 2. Linear Logic Index Access (`atData`)

Access elements via logical flattened index (as if the tensor was flattened).

```cpp
// Access the 5th element (logic sequence)
// If tensor is non-contiguous, it auto-calculates correct physical position
t.atData(4) = 1.0f;
```

## 3. Physical Index Access (`atData_`, `[]`)

Direct access to the underlying data array. **Only correct for contiguous tensors!**

```cpp
// Assume t is contiguous (IsContiguous() == true)
for (size_t i = 0; i < t.size(); ++i) {
    t[i] = 0.0f;      // Fastest
    // or
    t.atData_(i) = 0.0f; 
}
```

*   **Warning**: If `t` is a non-contiguous view (e.g., `slice` or `transpose` result), using this method traverses physical memory layout, which will NOT match logical element order and may access invalid data (though within buffer bounds).
