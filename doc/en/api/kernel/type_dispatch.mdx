# Type Dispatch

This document introduces the runtime type dispatch mechanism in the `yt::kernel` namespace.

## Overview

### What is Type Dispatch?

In YTensor, `YTensorBase` uses a string `dtype` (e.g., `"float32"`) to represent data types at runtime, while `YTensor<T, dim>` determines types at compile time. **Type Dispatch** maps runtime dtype strings to compile-time concrete types `T`, enabling the correct template function calls.

### Why Type Dispatch?

Suppose you need to write a function handling `YTensorBase`, but internally require access to concrete type data:

```cpp
void processData(const YTensorBase& tensor) {
    // tensor.dtype() could be "float32", "int64", etc.
    // How to call the correct template function based on dtype?
    
    // ❌ Traditional way: Verbose if-else
    if (tensor.dtype() == "float32") {
        const float* data = tensor.data<float>();
        // ...
    } else if (tensor.dtype() == "int64") {
        const int64_t* data = tensor.data<int64_t>();
        // ...
    }
    // Need to write for every type!
    
    // ✅ Using Type Dispatch: Concise and Efficient
    yt::kernel::dispatchOrThrow<yt::types::AllNumericTypes>(
        tensor.dtype(),
        [&]<typename T>() { // C++20 Generic Lambda
            const T* data = tensor.data<T>();
            // T is automatically deduced to the correct type
        },
        "processData"
    );
}
```

## Predefined Type Lists

YTensor predefines common type lists for convenience:

| Type List | Included Types |
| --- | --- |
| `StandardFloatTypes` | `float`, `double` |
| `StandardIntTypes` | `int8`, `int16`, `int32`, `int64`, `uint8`, `uint16`, `uint32`, `uint64` |
| `IntegerTypes` | Same as `StandardIntTypes` |
| `StandardNumericTypes` | Standard Floats + Standard Integers |
| `ExtendedFloatTypes` | `bfloat16`, `float16`, `float8_e5m2`, `float8_e4m3` etc. |
| `AllNumericTypes` | All of the above |

## `dispatch`

### Function Signature

```cpp
template<typename TypeListT, typename Func>
bool dispatch(const std::string& dtype, Func&& func);

template<typename TypeListT, typename Func>
void dispatchOrThrow(const std::string& dtype, Func&& func, const std::string& opName = "dispatch");
```

### Core Functionality

Based on the `dtype` string, finds a matching C++ type `T` in `TypeListT`, and calls the generic lambda `func.operator()<T>()`.

*   `dispatch`: Returns `true` if a match is found and executed; otherwise `false`.
*   `dispatchOrThrow`: Throws `std::runtime_error` if no match is found.

#### Automatic Nested Type Parsing

`dispatch` supports **automatic parsing of nested dtypes**. If the dtype string is nested (e.g., `"YTensorBase<float32>"`):

1.  First attempts to match the full dtype string exactly.
2.  If exact match fails, automatically calls `getBaseDtype()` to extract the base type.
3.  Uses the extracted base type (e.g., `"float32"`) for a second matching attempt.

This allows tensors returned by `matView()` to be correctly dispatched:

```cpp
YTensorBase tensor({3, 4, 5}, "float32");
auto matView = tensor.matView();  // dtype: "YTensorBase<float32>"

// Dispatch automatically parses "YTensorBase<float32>" -> "float32"
yt::kernel::dispatch<yt::types::AllNumericTypes>(matView.dtype(), [&]<typename T>() {
    std::cout << "Base type size: " << sizeof(T) << std::endl;  // Output: 4
});
```

### Usage Example

Writing a print function supporting all numeric types:

```cpp
#include "ytensor_kernel.hpp" // or type_dispatch.hpp

void printValue(const yt::YTensorBase& tensor) {
    // Define a generic lambda
    auto kernel = [&]<typename T>() {
        // T is now a concrete type (float, int, etc.)
        // Safe to access data via data<T>()
        const T* data = tensor.data<T>();
        std::cout << "Value: " << data[0] << std::endl;
    };

    // Dispatch!
    // Try to match tensor.dtype() in AllNumericTypes
    yt::kernel::dispatchOrThrow<yt::types::AllNumericTypes>(
        tensor.dtype(), 
        kernel, 
        "printValue"
    );
}
```

## `dispatch2`

### Function Signature

```cpp
template<typename SrcTypeList, typename DstTypeList, typename Func>
bool dispatch2(const std::string& srcDtype, const std::string& dstDtype, Func&& func);

template<typename SrcTypeList, typename DstTypeList, typename Func>
void dispatch2OrThrow(const std::string& srcDtype, const std::string& dstDtype, Func&& func, const std::string& opName = "dispatch2");
```

### Core Functionality

Double type dispatch. Used for scenarios requiring handling two different types simultaneously, such as casting or mixed-type arithmetic.
It attempts to match both `srcDtype` and `dstDtype`, invoking `func.operator()<SrcT, DstT>()`.

### Usage Example

Implementing a generic cast function:

```cpp
void castTensor(const yt::YTensorBase& src, yt::YTensorBase& dst) {
    auto kernel = [&]<typename SrcT, typename DstT>() {
        const SrcT* s = src.data<SrcT>();
        DstT* d = dst.data<DstT>();
        // Logic
        for(size_t i=0; i<src.size(); ++i) {
            d[i] = static_cast<DstT>(s[i]);
        }
    };

    yt::kernel::dispatch2OrThrow<yt::types::AllNumericTypes, yt::types::AllNumericTypes>(
        src.dtype(), 
        dst.dtype(), 
        kernel, 
        "castTensor"
    );
}
```

## Custom Trait Check

Sometimes just matching types isn't enough; you might need types that satisfy specific conditions (e.g., support addition).

```cpp
template<template<typename> class Trait, typename TypeListT, typename Func>
void dispatchWithTraitOrThrow(const std::string& dtype, Func&& func, const std::string& opName);
```

### Usage Example

Execute addition only for types supporting `operator+`:

```cpp
auto kernel = [&]<typename T>() {
    /* ... code instantiated only if T supports addition ... */
};

yt::kernel::dispatchWithTraitOrThrow<yt::concepts::HAVE_ADD, yt::types::AllNumericTypes>(
    tensor.dtype(),
    kernel,
    "addOperation"
);
```

---

## Best Practices

1.  **Use Generic Lambdas (C++20)**: With `[&]<typename T>() { ... }` syntax for cleaner code.
2.  **Select Appropriate TypeList**: Restrict to types you actually assume to support, reducing compile time and binary size (though instantiation is on-demand, failed matching attempts have overhead).
3.  **Error Handling**: For operations that must succeed, prefer `dispatchOrThrow` for clear error messages.
