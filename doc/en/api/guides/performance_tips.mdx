# Performance Tips
 
This guide introduces best practices for optimizing YTensor program performance.
 
## 1. Avoid Unnecessary Copies
 
-   **Use Views**: `slice`, `transpose`, `reshape` (usually) return views and do not copy data.
-   **Avoid `clone()`**: Only use it when you explicitly need a data copy.
-   **Reference Passing**: Pass `const YTensorBase&` or `const YTensor<T, D>&` as function parameters.
 
## 2. Leverage In-Place Operations
 
In-place operations avoid allocating new memory for results.
 
```cpp
// ❌ Creates new tensor 'c'
auto c = a + b;
 
// ✅ Reuse 'a' (if 'a' is no longer needed)
a += b;
```
 
## 3. Contiguous Memory Priority
 
Contiguous memory access is faster (cache-friendly) and enables SIMD optimizations.
 
-   **Construct Intelligently**: `zeros`, `ones` create contiguous tensors by default.
-   **Handling Non-Contiguous Tensors**: If you need to traverse a non-contiguous tensor (e.g., result of `transpose`) multiple times, consider calling `contiguous()` to copy it to a contiguous layout first.
 
## 4. Compile-Time Optimization
 
-   **Enable AVX2**: Ensure `-mavx2 -mfma` is enabled at compile time.
-   **Release Mode**: Always use `-O3` or Release mode for performance testing. Debug mode is significantly slower (due to boundary checks, assertion overhead).
 
## 5. Parallel Computing
 
YTensor automatically uses multithreading for many operations (like `matmul`, `foreach`).
 
-   **Note on Small Data**: For very small tensors, multithreading overhead might outweigh benefits.
 
## 6. Type Dispatch Optimization
 
When using `dispatch`:
 
-   **Specific List**: Try using `StandardFloatTypes` instead of `AllNumericTypes` if you only care about floats, to reduce template instantiation and lookup overhead.
 
## 7. Arithmetic Expressions
 
Although YTensor does not currently support operator fusion (Lazy Evaluation), for simple chained operations:
 
```cpp
// Will generate temporary tensors
auto d = a * b + c;
 
// Optimization: Break down into in-place operations
auto d = a * b;
d += c;
```
 
*Note: In future versions, operator overloading might introduce expression templates to auto-optimize this.*
 
## 8. IO Buffer Optimization
 
When saving large amounts of data:
 
-   Use `YTensorIO::alloc_buffer` to pre-allocate buffers if you know the data size roughly.
-   Use `compress=true` (default) to reduce disk I/O, though it consumes some CPU. For pure read/write speed (e.g., SSD), consider disabling compression.

---
 
## Checklist
 
- [ ] Is `-O3` enabled?
- [ ] Is unnecessary deep copying avoided?
- [ ] Are in-place operations used where possible?
- [ ] Is AVX2 instruction set enabled?
- [ ] Is data layout continuous during frequent access?
