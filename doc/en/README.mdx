# YTensor Documentation

> A modern, lightweight C++20 multi-dimensional tensor library — header-only, easy to integrate, suitable for research, competition, and engineering.

## Welcome to YTensor

YTensor is a pure header-only C++ tensor library, designed for scenarios requiring high-performance multi-dimensional array operations. It provides an interface similar to NumPy/PyTorch, but without any runtime dependencies; just include a single header file to use.

## Features

- 🧩 **Header-only**: Only `ytensor_single.hpp` is needed, zero third-party dependencies, just `#include` and use.
- 📐 **Arbitrary Dimensions**: Supports `shape`, `slice`, `transpose`, `reshape`, `permute`, and other common operations.
- ⚡ **Multi-functional**: Supports parallelism, broadcasting, basic math, common deep learning functions, multi-axis normalization, and SDPA / Flash Attention.
- 🛠️ **Easy to Extend**: Clean source code, easy for secondary development.
- 🗂️ **I/O Support**: Can save/load YTensor supported file formats.
- 🔢 **Wide Type Support**: Supports common data types, such as `float16`, `bfloat16`, etc.

## Quick Start

### Installation

See the [Installation Guide](./installation/installation.mdx) to learn how to integrate YTensor into your project.

### Basic Example

```cpp
#include "ytensor_single.hpp"

int main() {
    // Create a 3x4 float tensor
    yt::YTensor<float, 2> a(3, 4);
    
    // Create ones tensor
    auto b = yt::YTensor<float, 2>::ones(3, 4);
    
    // Access and modify elements
    a.at(1, 2) = 5.0f;
    
    // Perform operations
    auto c = a + b;
    
    // Output tensor
    std::cout << c << std::endl;
    
    return 0;
}
```

See the [Quick Start Guide](./api/guides/quick_start.mdx) for more examples.

## v0.13 Highlights

- `yt::function` is reorganized into `activation`, `normalization`, `ops`, and `loss`.
- Added `linear`, `rmsNorm`, `normalize`, `instanceNorm`, naive `batchNorm`, and `groupNorm`.
- Softmax is documented as an activation; non-element-wise ops no longer expose simplified `order` derivative APIs.
- SDPA now provides `MATH` and `FLASH_AVX2` backends with generic bool-mask, predicate-mask, and bias support.
