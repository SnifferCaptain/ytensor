# 类型注册

本文档介绍 `yt::infos` 命名空间中的类型注册系统。

## TypeRegItem 结构体

```cpp
struct TypeRegItem {
    std::string name;                              // 类型名称
    int32_t size;                                  // 字节大小
    std::function<std::string(const void*)> toString;  // 转字符串函数
    bool isPOD = true;                             // 是否为 POD 类型
    std::function<void(void*)> destructor = nullptr;   // 析构函数
    std::function<void(void*, const void*)> copyConstruct = nullptr;  // 拷贝构造
    std::function<void(void*)> defaultConstruct = nullptr;  // 默认构造
    
    // IO 序列化支持 (用于非 POD 类型)
    std::function<std::vector<char>(const void*)> serialize = nullptr;      // 对象 -> 字节序列
    std::function<void(void*, const char*, size_t)> deserialize = nullptr;  // 字节序列 -> 对象
};
```

## 类型注册表

```cpp
auto& getTypeRegistry();
```

返回全局类型注册表的引用。

### 注册类型

除了自动推导的 POD 类型外，YTensor 提供了 `registerType` 函数用于注册自定义类型。

#### 1. 基础注册 (自动 POD 检测)

```cpp
template<typename T>
void registerType(const std::string& typeName, 
                  std::function<std::string(const void*)> formatter = nullptr);
```

#### 2. 带 IO 支持的注册 (非 POD)

```cpp
template<typename T>
void registerType(const std::string& typeName,
                  std::function<std::string(const void*)> formatter,
                  std::function<std::vector<char>(const void*)> serialize,
                  std::function<void(void*, const char*, size_t)> deserialize);
```

此重载允许你为非 POD 类型（如 `std::string` 或自定义对象）提供序列化和反序列化函数，从而使这些类型能够被 `YTensorIO` 正确存储和加载。

## 浮点类型规范

YTensor 支持包括标准浮点与低精度浮点在内的多种浮点类型。
特别地，对于 BF16 类型，其舍入模式受全局配置影响（见 [常量](./constants.mdx)）。

## FloatSpec 通用浮点数模板

YTensor 提供 `FloatSpec` 模板类，支持在 IEEE 754 标准下生成任意指数位和尾数位配置的浮点数类型。

### 模板定义

```cpp
template <int SignBits, int ExpBits, int MantissaBits>
class FloatSpec;
```

| 参数 | 说明 |
| --- | --- |
| `SignBits` | 符号位数（0 或 1） |
| `ExpBits` | 指数位数 |
| `MantissaBits` | 尾数位数 |

### 预定义类型

这些类型已经在库中预定义，可直接使用：

| 类型名 | C++别名 | 定义 | 总位数 | 说明 |
| --- | --- | --- | --- | --- |
| `float16` | `yt::float16` | `FloatSpec<1, 5, 10>` | 16 | IEEE 754 半精度浮点数 |
| `bfloat16` | `yt::bfloat16` | `FloatSpec<1, 8, 7>` | 16 | Brain Floating Point |
| `float8_e5m2` | `yt::float8_e5m2` | `FloatSpec<1, 5, 2>` | 8 | Hopper 架构 FP8 |
| `float8_e4m3` | `yt::float8_e4m3` | `FloatSpec<1, 4, 3>` | 8 | Hopper 架构 FP8 |
| `float8_e8m0` | `yt::float8_e8m0` | `FloatSpec<0, 8, 0>` | 8 | 纯指数形式（Scaling Factor） |

### 使用示例

```cpp
#include "ytensor_types.hpp"

// 使用预定义类型
yt::bfloat16 bf = 3.14f;
float val = static_cast<float>(bf);

// 自定义浮点类型（TF32 格式，模拟）
using TF32 = yt::FloatSpec<1, 8, 10>;
TF32 tf = 2.5f;
```

## dtype 命名规范

YTensor 采用规范化的 dtype 命名格式，支持嵌套张量类型的表示：

### 基础类型名
使用 `getBaseTypeName<T>()` 获取基础 C++ 类型的名称：
- `float` → `"float32"`
- `double` → `"float64"`
- `int32_t` → `"int32"`
- 自定义类型 → `typeid(T).name()` 或注册名称

### 嵌套类型格式

| 张量类型 | dtype 格式 | 示例 |
| --- | --- | --- |
| `YTensor<T, dim>` | `"YTensor<scalar_dtype, dim>"` | `"YTensor<float32, 2>"` |
| `YTensorBase` | `"YTensorBase<inner_dtype>"` | `"YTensorBase<float32>"` |
| 多层嵌套 | 递归格式 | `"YTensor<YTensor<float32, 2>, 2>"` |

### dtype 解析函数

| 函数 | 功能 | 示例 |
| --- | --- | --- |
| `parseDtypeInner(dtype)` | 解析外层类型和内层内容 | `"YTensorBase<float32>"` → `{"YTensorBase", "float32"}` |
| `getBaseDtype(dtype)` | 递归提取最内层基础类型 | `"YTensor<YTensor<float32, 2>, 2>"` → `"float32"` |
| `getDtypeDim(dtype)` | 提取 YTensor 的维度 | `"YTensor<float32, 3>"` → `3` |
| `getDtypeScalar(dtype)` | 提取 YTensor 的 scalar 类型 | `"YTensor<float32, 2>"` → `"float32"` |
| `makeYTensorDtype(scalar, dim)` | 构建 YTensor dtype | `("float32", 2)` → `"YTensor<float32, 2>"` |
| `makeYTensorBaseDtype(inner)` | 构建 YTensorBase dtype | `("float32")` → `"YTensorBase<float32>"` |

### 使用示例

```cpp
#include "ytensor_types.hpp"

// 解析嵌套 dtype
std::string dtype = "YTensorBase<YTensor<float32, 2>>";
std::string baseDtype = yt::types::getBaseDtype(dtype);  // "float32"

// 构建 dtype
std::string ytensorDtype = yt::types::makeYTensorDtype("float32", 3);  // "YTensor<float32, 3>"

// 配合 dispatch 使用
YTensorBase tensor({2, 3, 4}, "float32");
auto matView = tensor.matView();  // dtype: "YTensorBase<float32>"

// dispatch 会自动解析嵌套类型
yt::kernel::dispatch<yt::types::AllNumericTypes>(matView.dtype(), [&]<typename T>() {
    // T 被解析为 float
});
```

## 非 POD 类型支持

对于非 POD 类型（如 `std::string`、自定义类），在使用前必须进行注册。注册时会自动推导析构函数和构造函数。

### 开启 IO 支持

默认情况下，`registerType` 仅注册基本的生命周期函数。如果需要支持文件 IO（即保存到 `.yt` 文件），必须显式提供 `serialize` 和 `deserialize` 函数。

```cpp
struct MyStruct {
    std::string data;
};

// 注册并提供 IO 支持
yt::types::registerType<MyStruct>(
    "MyStruct",
    // formatter
    [](const void* ptr) { return static_cast<const MyStruct*>(ptr)->data; },
    // serialize
    [](const void* ptr) -> std::vector<char> {
        const auto& s = static_cast<const MyStruct*>(ptr)->data;
        return std::vector<char>(s.begin(), s.end());
    },
    // deserialize
    [](void* ptr, const char* data, size_t len) {
        new (ptr) MyStruct{std::string(data, len)};
    }
);
```

:::info
**存储格式**
对于注册了序列化函数的类型，YTensorIO 会使用 `map` 格式存储，确保能够正确处理变长数据和对象状态。
:::
